# ระบบจัดการองค์กร
## Organization Management System

ระบบจัดการองค์กรครบวงจรสำหรับจัดการสมาชิก ฝ่าย/แผนก งาน/กิจกรรม และปัญหา/ข้อเสนอแนะ

## ฟีเจอร์

### ✅ ฟีเจอร์หลัก

1. **จัดการสมาชิก**
   - เพิ่ม/แก้ไข/ลบสมาชิก
   - กำหนดหัวหน้าฝ่าย
   - จัดการข้อมูลส่วนตัว (ชื่อ, อีเมล, โทรศัพท์, ตำแหน่ง)

2. **จัดการฝ่าย/แผนก**
   - เพิ่ม/แก้ไข/ลบฝ่าย
   - กำหนดหัวหน้าฝ่าย
   - คอมเมนต์ฝ่าย (ข้อเสนอแนะ, ปัญหา, การปรับปรุง)

3. **ระบบงาน/กิจกรรม**
   - เพิ่มจำนวนงาน/กิจกรรมให้สมาชิก
   - ตรวจสอบสมาชิกที่ทำงานมากที่สุด
   - ตรวจสอบสมาชิกที่ยังไม่เคยทำงาน

4. **ปัญหาและข้อเสนอแนะ**
   - แจ้งปัญหาที่พบ
   - เสนอข้อเสนอแนะ
   - ติดตามสถานะปัญหา (รอดำเนินการ, กำลังดำเนินการ, แก้ไขแล้ว)

5. **สถิติ**
   - จำนวนสมาชิกทั้งหมด
   - จำนวนฝ่าย
   - สมาชิกที่มีงาน/ไม่มีงาน
   - จำนวนงานทั้งหมด
   - จำนวนหัวหน้าฝ่าย

## การติดตั้ง

### 1. ติดตั้ง Dependencies

```bash
npm install
```

### 2. เริ่มต้นเซิร์ฟเวอร์

```bash
npm start
```

### 3. เข้าถึงเว็บไซต์

เปิดเบราว์เซอร์และไปที่:
```
http://localhost:3000
```

## วิธีใช้งาน

### เพิ่มสมาชิก
1. ไปที่แท็บ "สมาชิก"
2. กรอกข้อมูลสมาชิก
3. เลือกฝ่าย (ถ้ามี)
4. ตรวจสอบช่อง "เป็นหัวหน้าฝ่าย" ถ้าต้องการ
5. กดปุ่ม "บันทึกสมาชิก"

### เพิ่มฝ่าย
1. ไปที่แท็บ "ฝ่าย"
2. กรอกชื่อฝ่าย
3. เลือกหัวหน้าฝ่าย (ถ้ามี)
4. กรอกคำอธิบาย (ไม่บังคับ)
5. กดปุ่ม "บันทึกฝ่าย"

### เพิ่มงาน/กิจกรรม
1. ไปที่แท็บ "งาน/กิจกรรม"
2. เลือกสมาชิก
3. กดปุ่ม "เพิ่มจำนวนงาน"
   หรือ
   - ไปที่แท็บ "สมาชิก"
   - คลิกปุ่ม "เพิ่มงาน" ในสมาชิกที่ต้องการ

### แจ้งปัญหา/ข้อเสนอแนะ
1. ไปที่แท็บ "ปัญหา/ข้อเสนอแนะ"
2. เลือกประเภท (ปัญหาที่พบ หรือ ข้อเสนอแนะ)
3. กรอกหัวข้อและรายละเอียด
4. กรอกชื่อผู้แจ้ง (ไม่บังคับ)
5. กดปุ่ม "ส่ง"

### คอมเมนต์ฝ่าย
1. ไปที่แท็บ "ฝ่าย"
2. คลิกปุ่ม "คอมเมนต์" ในฝ่ายที่ต้องการ
3. เลือกประเภท (ข้อเสนอแนะ, ปัญหาที่พบ, การปรับปรุง)
4. กรอกข้อความ
5. กดปุ่ม "ส่ง"

### ตรวจสอบสถิติ
- ดูสถิติทั่วไปที่ด้านบนของหน้าเว็บ
- ดูรายละเอียดในแท็บ "งาน/กิจกรรม"
- ใช้ตัวกรองในแท็บ "สมาชิก" เพื่อค้นหา

## โครงสร้างโปรเจ็กต์

```
.
├── server.js          # เว็บเซิร์ฟเวอร์และ API
├── package.json       # Dependencies
├── data/              # ข้อมูล (สร้างอัตโนมัติ)
│   ├── members.json
│   ├── departments.json
│   ├── department-comments.json
│   └── issues.json
└── public/            # ไฟล์ Frontend
    ├── index.html     # หน้าเว็บหลัก
    └── app.js         # JavaScript Frontend
```

## API Endpoints

### สมาชิก (Members)
- `GET /api/members` - ดึงรายชื่อสมาชิกทั้งหมด
- `GET /api/members/:id` - ดึงข้อมูลสมาชิก
- `POST /api/members` - เพิ่มสมาชิกใหม่
- `PUT /api/members/:id` - อัปเดตข้อมูลสมาชิก
- `DELETE /api/members/:id` - ลบสมาชิก
- `POST /api/members/:id/work` - เพิ่มจำนวนงาน

### ฝ่าย (Departments)
- `GET /api/departments` - ดึงรายชื่อฝ่ายทั้งหมด
- `POST /api/departments` - เพิ่มฝ่ายใหม่
- `PUT /api/departments/:id` - อัปเดตฝ่าย
- `DELETE /api/departments/:id` - ลบฝ่าย
- `GET /api/departments/:id/comments` - ดึงคอมเมนต์ฝ่าย
- `POST /api/departments/:id/comments` - เพิ่มคอมเมนต์ฝ่าย

### ปัญหา/ข้อเสนอแนะ (Issues)
- `GET /api/issues` - ดึงปัญหาทั้งหมด
- `POST /api/issues` - เพิ่มปัญหา/ข้อเสนอแนะ
- `PUT /api/issues/:id` - อัปเดตสถานะปัญหา

### สถิติ (Statistics)
- `GET /api/stats` - ดึงสถิติทั้งหมด

## หมายเหตุ

- ข้อมูลทั้งหมดเก็บในรูปแบบ JSON ในโฟลเดอร์ `data/`
- ระบบจะสร้างโฟลเดอร์ `data/` อัตโนมัติเมื่อเริ่มต้น
- สามารถเข้าถึงได้จากหลายเบราว์เซอร์พร้อมกัน
- ไม่มีระบบ Authentication (ทุกคนเข้าถึงได้)

## การพัฒนาเพิ่มเติม

สามารถเพิ่มฟีเจอร์ต่อไปนี้ได้:
- ระบบ Authentication/Login
- Export ข้อมูลเป็น Excel/PDF
- แจ้งเตือนผ่าน Email
- Dashboard แสดงกราฟ
- ระบบ Backup/Restore
- การแชร์ข้อมูลระหว่างเครื่อง
